Branches

Branches são usadas para desenvolver funcionalidades isoladamente umas das outras. A branch principal é a master, onde o código está sempre pronto para produção. As branches de desenvolvimento são criadas a partir da master e, quando a funcionalidade estiver pronta, ela é mesclada de volta para a master.

Branching

Branching é o processo de criar uma nova branch a partir de uma branch existente. Isso permite que os desenvolvedores trabalhem em funcionalidades isoladamente, sem afetar o código principal. Quando a funcionalidade estiver pronta, ela pode ser mesclada de volta para a branch principal.

Merge

Merge é o processo de mesclar uma branch de desenvolvimento de volta para a branch principal. Isso é feito quando a funcionalidade está pronta e testada, e pode ser integrada ao código principal. O merge pode ser feito usando o comando git merge, que combina as mudanças da branch de desenvolvimento com a branch principal.
